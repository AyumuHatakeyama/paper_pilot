# 手動リグレッションチェックリスト

自動テストでカバーしていない部分（実際のLINEアカウント・Claude API・Supabase本番DB・pg_cronの
タイミングに依存する部分）を対象にした、リリース前の手動確認項目。

## 自動テストでカバー済みの範囲（このチェックリストの対象外）

以下は自動テストがあるため、通常はこのチェックリストで手動確認しなくてよい。
ロジックを変更した場合は、まず該当テストを実行してグリーンであることを確認すること。

- `supabase/functions/_shared/line-client_test.ts` — LINE署名検証（正規/改ざん/鍵不一致）
- `supabase/functions/_shared/jst-date_test.ts` — JST日付変換（00:00〜08:59 JSTの境界値含む）
- `supabase/functions/webhook-line/print-flow_test.ts` — 解析結果の返信文生成、ToDo確認プロンプトのトリガー条件・ラベリング・postback data組み立て
- `supabase/functions/webhook-line/bulk-register-flow_test.ts` — 予定一括登録の一覧テキスト・確認テキスト生成
- `supabase/functions/cron-notify/logic_test.ts` — JST時刻変換、日数計算、締切リマインド/週次ダイジェストの本文生成、曜日指定時のダイジェスト送信可否判定
- `web/src/lib/date-utils.test.ts` — 「あと◯日」計算（タイムゾーンをまたぐ境界値含む）

実行方法:
```
# Edge Functions（Deno）
deno test --allow-env --env-file=supabase/functions/.env.test supabase/functions

# Web（Vitest）
cd web && npm test
```

## 1. LINE Bot — 画像解析（P1 / P3）

- [ ] プリント画像を送信 → 「指示あり/なし」のQuick Replyが届く
- [ ] 「指示なし」を選択 → 解析結果（対象者・種別・内容・日程一覧）が返信される
- [ ] 「指示あり」を選択 → 指示テキストの入力を促され、入力後に指示を反映した解析結果が返信される
- [ ] 表形式のプリント（複数日程が縦に並ぶもの）→ 各行が正しく個別のイベントとして抽出される
- [ ] 締切・日程ありのプリント → 解析結果に続けてToDo追加のQuick Reply確認メッセージが届く
- [ ] 締切・日程なしのプリント（すべて過去日、is_deadlineもfalse）→ ToDo確認メッセージが出ない
- [ ] 「ToDoに追加＋リマインドON」選択 → Web側のプリント詳細画面にToDoが表示され、リマインド🔔アイコンが付く
- [ ] 「ToDoに追加（リマインドなし）」選択 → ToDoは追加されるが🔔は付かない
- [ ] 「ToDo不要（記録のみ）」選択 → todosテーブルにレコードが作られない
- [ ] 1枚に複数日程があるプリントで「ToDoに追加」→ 該当する全イベントがまとめてToDo化される
- [ ] 画像アップロード失敗・不正なファイル形式 → エラーメッセージが返る（アプリが落ちない）
- [ ] 「ToDoに追加」ボタンを連続で2回タップ（二重送信）→ todosが重複作成されない（2回目は「既にToDoに追加済みです」等になる）
- [ ] 深夜0時台〜朝9時前（JST）にプリントを送信 → 当日締切のプリントで正しくToDo確認プロンプトが出る（前日付として誤判定されない）

## 2. LINE Bot — 予定一括登録

- [ ] リッチメニュー「予定登録」→ 入力促進メッセージが届く
- [ ] 複数行のテキスト（例: `8/12-14 学童お弁当持参`）→ 日数分に展開されて確認テキストが届く
- [ ] 情報不足なテキスト → Claudeからの逆質問が届き、回答後に確認テキストへ進む
- [ ] 確認テキストで「はい」→ print_eventsに登録され、todosにはtodo_enabled=falseで自動記録される（ToDo確認プロンプトは出ない）
- [ ] 確認テキストで「いいえ」→ 何も登録されない
- [ ] 確認待ち中に無関係なテキストを送る → ボタン操作を促すメッセージが返る（再パースされない）
- [ ] セッションTTL（5分）経過後に操作 → 「有効期限が切れました」と案内される

## 3. LINE Bot — ブック登録（P5）

- [ ] リッチメニュー「ブックで登録」→ 画像収集開始メッセージが届く
- [ ] 複数画像を送信 → 送信するたびに枚数カウントが返信される
- [ ] 上限枚数（10枚）到達 → 自動でタイトル入力待ちに遷移する
- [ ] 「完了」送信 → タイトル入力を促される
- [ ] タイトル入力 → 各ページが順に解析され、成功/失敗件数が最後にまとめて返信される
- [ ] ブック経由の登録 → todosにtodo_enabled=falseで自動記録され、ToDo確認プロンプトは出ない
- [ ] Web「ブック詳細」画面で該当ブックの全ページが表示される
- [ ] 「キャンセル」送信 → セッションが破棄される

## 4. LINE Bot — チャットQ&A

- [ ] 「今週の締切は？」等の質問 → 現在有効なプリントを踏まえた回答が返る
- [ ] 有効なプリントが0件のとき質問 → 「現在有効なプリントはありません」と案内される

## 5. 通知機能（cron-notify）

pg_cronのタイミングに依存するため、`notification_settings.send_time`を直近の時刻に一時変更して確認する。

- [ ] 新規ユーザーの初回LINEインタラクション後、`notification_settings`にdaily/22:00/全ONの初期行が自動作成される
- [ ] `send_time`を現在時刻の直後（次の毎時0分）に変更 → その時刻にLINEへ通知が届く
- [ ] 締切当日のToDo（reminder_enabled=true）がある状態で通知時刻を迎える → 締切リマインドが届く
- [ ] 「要準備」カテゴリのToDoで締切3日前 → リマインドが届く（提出のみ/未設定は届かない）
- [ ] 「提出のみ」カテゴリのToDoで締切1日前 → リマインドが届く
- [ ] 締切リマインドと週次ダイジェストが同時刻に発生する条件 → 1通に統合されて届く（別々の2通にならない）
- [ ] `digest_enabled`をOFFにして通知時刻を迎える → ダイジェストは届かず、リマインドのみ届く（該当あれば）
- [ ] `reminder_enabled`をOFFにして通知時刻を迎える → リマインドは届かず、ダイジェストのみ届く
- [ ] 対象0件・`send_when_empty=true`の状態で通知時刻を迎える → 空状態メッセージ（「未処理のToDoはありません」等）付きで届く
- [ ] Web設定画面で「毎日→特定曜日」に変更 → 翌日以降その曜日以外の時刻には届かない（ダイジェストのみ。締切リマインドは曜日指定でも対象日には届く）
- [ ] Web設定画面で送信時刻を変更 → 変更後の時刻に届く（変更前の時刻には届かない）
- [ ] `frequency='weekly'`かつ`reminder_enabled=true`の状態で、指定曜日以外の日に締切当日のToDoがある → その曜日でも締切リマインドは届く（ダイジェストは届かない）

## 6. Web — カレンダー画面（`/`）

- [ ] 月送り（‹ ›）で予定・締切・ToDoのドット表示が正しい月に切り替わる
- [ ] 日付セルをクリック → その日の予定・締切・ToDoが下に表示される
- [ ] 未完了ToDoの期限が3日以内（超過含む）の日 → ⚠アイコン＋赤字で強調される
- [ ] カレンダー・一覧上のToDoチェックボックスをON → 一覧から即座に消える（未完了のみ表示のため）
- [ ] 「＋」ボタン → 手動予定追加画面に遷移する（選択日から遷移した場合は日付が引き継がれる）

## 7. Web — プリント詳細画面（`/prints/[id]`）

- [ ] 日程一覧・元画像・内容が正しく表示される
- [ ] 紐づくToDoの完了/未完了をトグルできる（完了後も一覧に残り、取り消し線表示になる）
- [ ] ToDoトグルの更新に失敗した場合（例: ネットワーク切断） → エラーアラートが表示される（チェック状態が無言で戻るだけにならない）
- [ ] 「アーカイブ」→ 確認ダイアログ後、トップへ遷移しアーカイブ画面に表示されるようになる
- [ ] 画像URLが無効/読み込み失敗 → 画像セクションが表示されない（エラーで画面全体が壊れない）
- [ ] ToDoの取得だけが失敗する状況（該当プリントのtodosクエリのみ失敗）でも、プリント本体・日程一覧は正常に表示される

## 8. Web — 設定画面（`/settings`）

- [ ] 既存設定が無い新規ユーザーでアクセス → デフォルト値（毎日・22:00・全ON）が表示される
- [ ] 「曜日を指定」選択時のみ曜日ボタンが表示される
- [ ] 保存後、DBの`notification_settings`が更新されている（`weekly_day`は`daily`選択時にnullへ戻ることを確認）
- [ ] 設定の読み込みに失敗する状況（例: ネットワーク切断）→ フォームではなく「読み込みに失敗しました」＋再読み込みボタンが表示される（デフォルト値のフォームが表示されて誤って保存できてしまわないこと）

## 9. Web — アーカイブ画面（`/archive`）

- [ ] アーカイブ済みプリントが一覧表示される
- [ ] 「↩」で有効に戻すと一覧から消え、トップに再表示される
- [ ] 締切から30日経過したプリントが自動アーカイブされる（pg_cronジョブ`auto-archive-prints`。マイグレーション管理外のため、DBを直接見て確認する）

## 10. Web — 手動イベント追加・編集・削除

- [ ] 新規追加（日付・時間・タイトル・締切フラグ・対象者・補足）→ カレンダーに反映される
- [ ] 編集画面でLINE/プリント由来のイベントIDを直接開いた場合 → トップへリダイレクトされる
- [ ] 削除 → カレンダーから消える

## 11. 認可・アクセス制御

- [ ] `NEXT_PUBLIC_ALLOWED_LINE_USER_IDS`に含まれないLINEユーザーでLIFFアクセス → 「アクセス権限がありません」画面が表示される
- [ ] LIFF_ID未設定のローカル開発環境 → dev-userとして認証をスキップして画面が使える
